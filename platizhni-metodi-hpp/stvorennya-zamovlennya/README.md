# Створення замовлення

Метод використовується для створення нового замовлення з метою ініціації платежу через **Hosted Payment Page (HPP)** або **мобільний застосунок банку**.

Після успішного створення замовлення система генерує **платіжне посилання (redirectUrl)**, яке може містити:

* посилання на HPP-сторінку
* посилання для відкриття мобільного застосунку банку, через який клієнт може зручно завершити оплату.

Метод підтримує створення замовлень для двох основних типів операцій:

* **PURCHASE** - оплата товару або послуги;
* **A2A** - переказ між рахунками (_account-to-account_).

**Типи переходів на оплату**

* **REDIRECT** (за замовчуванням) - клієнт перенаправляється на сторінку HPP для завершення платежу;
* **BANK\_LINK** - клієнт перенаправляється безпосередньо до мобільного застосунку Альянс Банку або до пріоритетного банку, вказаного в параметрі `priorityBankCode`, для виконання A2A-платежу.

**Додаткові параметри**

* **priorityBankCode** - задає код пріоритетного банку, який буде враховано при формуванні `redirectUrl`.
* **generateQrNbu** - вмикає генерацію QR-коду у форматі НБУ, який можна відобразити клієнту для оплати через мобільний застосунок банку.

У результаті виклику мерчант отримує ідентифікатор замовлення, статус, посилання на оплату, а при потребі - QR-код НБУ.&#x20;

Це дозволяє гнучко реалізувати як веб-, так і мобільні сценарії оплати в межах єдиної інтеграції.
